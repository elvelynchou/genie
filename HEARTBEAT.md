# GenieBot 赛博脉搏协议 (HEARTBEAT)

## 1. 任务清单与执行频率
1.  **[财经监控] (finance_monitor)**
    -   **频率**：每 30 分钟。
    -   **动态逻辑**：检查当前市场关键指标（如 BTC 价格或主要股指）。如果 10 分钟内波动率 > 2%，优先触发。
    -   **静默期**：北京时间 23:30 - 07:30 停止自动执行（除非手动触发）。

2.  **[科技趋势扫描] (daily_report)**
    -   **频率**：每日 10:00 (北京时间)。
    -   **容错逻辑**：如果 10:00 执行失败，在后续每次心跳中检测并尝试补跑，直到今日成功一次为止。

3.  **[记忆巩固/梦境阶段] (dreamer)**
    -   **频率**：每日 03:00 (北京时间)。
    -   **逻辑**：在系统最闲置时运行，提炼昨日的 Data 层事实为 Strategy 层策略。

4.  **[系统自检] (sys_check)**
    -   **频率**：每次心跳 (10 分钟)。
    -   **内容**：检查 `/etc/myapp/genie/logs/` 大小，如果 > 500MB 则触发清理；检查 Redis 连通性。

## 2. 执行逻辑 (Logic Layer)
-   **Step 1: 环境感知**
    -   读取系统当前时间（北京时间）。
    -   读取 `src/agents/task_status.json` 获取各任务 `last_run_time`。
-   **Step 2: 决策判断**
    -   对比“当前时间”与“上次执行时间”。
    -   如果满足频率条件 (Now.hour == 3 且今日未跑 ➔ dreamer) ➔ 生成待执行任务列表。
-   **Step 3: 任务分发**
    -   依次调用对应的 Agent 执行任务。
    -   更新 `src/agents/task_status.json` 中的时间戳。
-   **Step 4: 退出机制**
    -   如果没有任何任务达到触发条件，回复 `HEARTBEAT_OK`。

## 3. 运行规范
-   心跳唤醒应使用 **Gemini Flash** 模型以控制成本。
-   所有心跳决策过程必须记录在 `logs/heartbeat.log` 中。
